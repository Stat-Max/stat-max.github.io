CSS position static · relative · absolute · fixed · sticky

static 기본값 — 일반 흐름(normal flow) 그대로

모든 요소의 기본값입니다. top / left / right / bottom 속성이 무시됩니다. 문서 흐름에 따라 순서대로 배치됩니다.

형제 요소 A
position: static (기본값) — 흐름대로
형제 요소 B
position: static;  /* top/left 등 좌표 속성 무효 */

relative 원래 자리 기준으로 이동

자기 원래 위치를 기준으로 top / left 등으로 이동합니다. 원래 자리는 비워진 채 유지되어 형제 요소에 영향을 주지 않습니다. 자식 absolute 요소의 기준점(containing block)이 됩니다.

형제 요소 A
position: relative  ·  top: 16px, left: 24px
형제 요소 B — 원래 자리는 그대로 유지됨
position: relative;
top: 16px;
left: 24px;  /* 원래 위치에서 16px 아래, 24px 오른쪽으로 이동 */

absolute 가장 가까운 positioned 조상 기준

일반 흐름에서 완전히 빠져나옵니다. position: relative/absolute/fixed/sticky인 가장 가까운 조상을 기준으로 좌표가 결정됩니다. 조상이 없으면 <html>(뷰포트)이 기준이 됩니다.

형제 요소 A — absolute 박스 자리 없음
형제 요소 B
absolute
top:12 right:12 
/* 부모에 position:relative 필요 */
position: absolute;
top: 12px;
right: 12px;

fixed 뷰포트(화면) 기준으로 고정

스크롤해도 항상 같은 위치에 고정됩니다. 기준은 뷰포트(브라우저 창)입니다. 일반 흐름에서 빠져나오며, 네비게이션 바·플로팅 버튼에 주로 사용합니다. → 오른쪽 하단의 초록 버튼이 실제 fixed 예시입니다.

콘텐츠 영역 (스크롤해도 fixed 요소는 화면에 고정)

콘텐츠 영역

fixed
bottom:12 right:12
position: fixed;
bottom: 20px;
right: 20px;  /* 뷰포트 기준, 스크롤 무관하게 고정 */

sticky 스크롤 중 임계점에서 고정

평소에는 relative처럼 동작하다가, 스크롤이 지정한 임계점(예: top: 0)에 닿는 순간 fixed처럼 고정됩니다. 부모 컨테이너를 벗어나면 다시 흘러갑니다. 섹션 헤더·테이블 열 머리에 자주 사용합니다. 아래 박스를 직접 스크롤해 보세요.
